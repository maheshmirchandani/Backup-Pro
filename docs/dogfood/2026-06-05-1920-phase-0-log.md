> **Gate to close Phase 0:** 50 cumulative backup runs without a data-loss bug, plus weekly verify clean, over a 2-week window. Target close: **2026-06-19**.
>
> **Re-entry to a Claude session triggers on:** Sev1 (data loss), Sev2 (verify failure / hash mismatch / atomic-gate fire), friction noticed twice across separate runs, or the 2-week mark.

## Sev1 / Sev2 incidents

> **Sev1:** data loss, source unlink without verify, `crashed_resumed` finalize that ate live data, atomic-gate fired but source was deleted anyway.
>
> **Sev2:** verify failure, hash mismatch on a known-good USB, manifest tamper trip that is not operator-caused, atomic-gate fired in a copy-mode run.
>
> If a Sev1 happens, stop dogfood, capture `make debug-bundle RUN=<run-id> USB=/Volumes/<USB>` from project root, and start a new Claude session with the tarball attached.
